## Context

既有 `star-pyramid` 能力已定義由窄到寬的置中金字塔。本變更在相同高度 \(n\) 下新增「倒立」變體：第 1 行最寬、第 \(n\) 行最窄，與原金字塔行順序相反。實作應與現有驗證與輸出風格一致（Java/Spring 專案慣例、DTO/服務分層等）。

## Goals / Non-Goals

**Goals:**

- 以明確公式產生倒立金字塔各行（空白與星號數與 \(n\)、行號 \(i\) 對應可測）。
- 高度驗證與錯誤語意與現有金字塔一致。
- 提供與現有一致的「逐行輸出」途徑，不竄改每行字串內容。

**Non-Goals:**

- 不要求單一 API 同時回傳正立與倒立（除非產品後續明定）；規格以「能產生並可輸出倒立行序列」為主。
- 不變更既有正立金字塔規格或行為。

## Decisions

1. **行公式（1-based 第 \(i\) 行）**  
   倒立第 \(i\) 行對應正立第 \(k = n - i + 1\) 行。沿用正立規則可得：左側空白數為 \(i - 1\)，星號數為 \(2n - 2i + 1\)。行字串為 `" ".repeat(i - 1) + "*".repeat(2n - 2i + 1)\)。  
   *理由：* 與現有 `star-pyramid` 數學關係一致，僅為行序反轉，便於測試對照與手動驗算。

2. **程式結構**  
   在核心邏輯層新增「產生倒立行列表」的函式或對等方法；驗證 \(n\) 可抽成共用或重複呼叫與正立相同規則（依專案現況擇一），避免兩套不一致的邊界條件。

3. **輸出**  
   與 `Print pyramid using generated lines` 相同精神：由外層依序寫出各行（換行分隔），不修改行內字元。

*曾考慮* 以「產生正立後 `reverse()`」實作倒立：語意等價且可減少公式重複，但若正立與倒立分屬不同入口，直接依上式產生亦清晰；實作者可擇一並以規格中的情境測試鎖定行為。

## Risks / Trade-offs

- **[Risk]** 公式與正立混用時行號 1-based 說明不清 → **Mitigation**：規格與測試涵蓋 \(n=1,2,3,5\) 等情境，與正立範例可手動對照。
- **[Trade-off]** 共用驗證 vs. 複製貼上：前者較 DRY，後者較少耦合；以專案既有風格為準。

## Migration Plan

- 純新增能力：部署新版本即可；無資料遷移。若 API 路徑新增，舊客戶端不受影響。
- **Rollback**：還原至不含倒立邏輯之版本；無額外 rollback 步驟。
