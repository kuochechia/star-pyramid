## Context

此變更新增一個教學/示範用、可重用且可測試的「星星金字塔」輸出能力。需求核心是：給定高度 \(n\)，產生 \(n\) 行字串；第 \(i\) 行（1-based）包含左側空白 + 星號 \(2i-1\) + 右側（可省略）空白，使其視覺上置中。

專案目前尚未有既有 OpenSpec capabilities（`openspec/specs/` 為空）。因此本次以新增能力 `star-pyramid` 為主，並避免對專案既有結構做假設：專案以新增一支 API 的形式呈現，以「URL後面帶高度參數」方式整合；將 I/O 與核心邏輯解耦，方便後續接任何介面。

主要約束：

- 產出必須可被單元測試穩定驗證（避免把行為綁死在 `console.log` / `stdout`）。
- 參數驗證要明確（例如：高度需為正整數；不合法輸入要有一致策略）。
- 以最小依賴實作（此能力不需要引入外部套件）。

## Goals / Non-Goals

**Goals:**

- 提供核心 API：給定高度 \(n\) 回傳金字塔每一行字串（例如 `string[]` 或 iterator），以利測試與重用。
- 定義清楚的輸出格式（置中、每行星號數規則、行尾換行策略）。
- 提供薄的整合層：將核心結果輸出到終端（專案入口為 API URL）；整合層不承擔排版邏輯。
- 對不合法輸入提供一致且可預期的行為（回傳錯誤/拋例外/顯示用法）。

**Non-Goals:**

- 不在本次設計中加入多種樣式（如倒金字塔、菱形、不同填充字元、左/右對齊等）——先把最基本的置中金字塔規格固定。
- 不引入互動式 UI（例如持續提示輸入、圖形介面）。
- 不做效能微調（高度通常很小；\(O(n^2)\) 字串組裝已足夠）。

## Decisions

- **決策：核心邏輯與輸出分離（pure generator + thin I/O layer）**
  - **方案**：實作一個純函式（例如 `generateStarPyramidLines(n): string[]`），只負責回傳逐行字串。
  - **理由**：可測試性最佳；輸出端以 Web 回應；避免在測試中攔截 `stdout`。
  - **替代方案**：直接在函式內 `console.log` / `stdout.write`。
    - **不選原因**：副作用難測；難以重用；錯誤處理/格式化耦合。

- **決策：高度參數採「正整數」且錯誤處理一致**
  - **方案**：將驗證集中在核心 API 入口（或共用 validator）；對 `n <= 0`、非整數、非數字字串等，回傳可診斷錯誤（例如拋出帶訊息的例外，或回傳 `Result` 型態，視專案慣例）。
  - **理由**：讓行為可預期；避免測試各自做不同驗證造成不一致。
  - **替代方案**：自動取整（floor/round）或把無效視為 0 行。
    - **不選原因**：隱式行為容易造成困惑；規格不清楚時會讓測試與教學結果不一致。

- **決策：輸出格式以「左側空白 + 星號」為準，行尾不補右側空白**
  - **方案**：第 \(i\) 行左側空白數為 \(n-i\)，星號數為 \(2i-1\)，回傳字串為 `" ".repeat(n-i) + "*".repeat(2i-1)`；最後由輸出層決定是否加入換行/如何 join。
  - **理由**：視覺置中效果相同；字串更精簡；測試更直觀。
  - **替代方案**：同時補齊右側空白到固定寬度 \(2n-1\)。
    - **不選原因**：多餘空白對終端顯示無益；測試字串更冗長，且容易因尾端空白被編輯器/格式化工具影響。

- **決策：規格示例驅動測試**
  - **方案**：在 specs 中提供範例（例如 \(n=1,2,3,5\)），並在測試中逐行比對陣列。
  - **理由**：需求清晰；回歸測試穩定；教學價值高。
  - **替代方案**：只測行數與每行星號數。
    - **不選原因**：無法防止置中/空白錯誤；對輸出格式的保護不足。

## Risks / Trade-offs

- **[風險] 不同平台/字型導致「置中」視覺差異** → **緩解**：規格以「字元數」定義置中（左側空白數與星號數），而非以像素/終端渲染效果定義。
- **[取捨] 先不支援多種樣式可能讓使用者期待落空** → **緩解**：在 specs 明確宣告目前只支援基本置中金字塔；未來擴充可新增參數或新增 capability（例如 `pyramid-styles`）。
- **[風險] 專案若沒有明確 CLI 入口，整合位置不明** → **緩解**：以模組/函式作為最小可交付物；若偵測到既有入口，再以最小改動接入。
